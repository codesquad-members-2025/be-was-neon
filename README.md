# be-was-2024
코드스쿼드 백엔드 교육용 WAS 2024 개정판

공부 정리한 내용은 github Wiki에 작성
## 웹 서버 1단계 - index.html 응답
Thread 기반을 Concurrent 패키지를 사용하도록 변경.  
Request Line에서 path를 분리하여 정적 파일의 내용을 반환하도록 함.

## 웹 서버 2단계 - 다양한 컨텐츠 타입 지원
RequestHandler의 로직을 크게 입력, 출력으로 나누었다.  
입력을 읽어, RequestHeader로 변환해서 쉽게 원하는 header를 찾을 수 있는 로직을 추가했다.  
파일의 확장자에 따라 response의 Content-Type을 설정하도록 했다.  
예외가 발생하면 우선 RequestHandler로 던져 처리할 수 있도록 했다.  

## 웹 서버 3단계 - GET으로 회원가입
Get으로 회원가입을 요청할 수 있도록 했다.  
요청을 받을 때 요청에 따라, 적절한 Handler를 사용하도록 Dispatcher를 구현했다.  
Dispatcher는 요청에 맞는 Handler를 반환하고, Handler 인터페이스를 사용하여 다형성을 구현했다.  
주요 기능에 따라 Handler를 나누었다.  

## 웹 서버 4단계 - POST로 회원 가입
Get으로 받던 회원가입 요청을 Post로 변경하고, 요청의 body에 데이터를 담아 보내도록 수정했다.  
이제는 body의 내용도 필요하기 때문에, body의 내용과 헤더의 내용을 나눠서 읽도록 했다.  
기존의 RequestHeader와 body의 내용을 합쳐 Request라는 클래스로 관리하도록 수정했다.  
나머지 로직은 동일하게 유지했다.   
추가로, 중복되는 매직 스트링을 상수로 관리하도록 했다.  

## 웹 서버 5단계 - 쿠키를 이용한 로그인
로그인 성공시 세션 ID를 생성해서, 해당 세션을 저장하고, 세션 ID를 쿠키에 넣도록 했다.  
로그아웃 시에는 해당 세션 ID를 가져와서, 해당 세션을 무효화(제거)하고, 쿠키도 제거하도록 했다.  
추가로 예외처리도 다시 리팩토링 했는데, HttpException 커스텀 예외를 만들고, 예외 발생시 해당 커스텀 예외를 던지도록 했다.  
그리고 RequestHandler에서 전역적으로 예외를 처리하도록 했다.  
추후에 서비스 로직의 예외는 따로 페이지가 생기면, 해당 페이지로 리다이렉트 해주도록 변경하면 좋을 것 같다.
